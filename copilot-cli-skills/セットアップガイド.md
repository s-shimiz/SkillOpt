| 必要なもの | 確認方法 | 備考 |

## 2. インストール（コピーするだけ）

スキルは「特定のフォルダに置く」だけで使えるようになります。

### 手順

1. このフォルダ（`copilot-cli-skills`）の中の **5 つのスキルフォルダ**を用意します。
   - `skillopt-sleep`, `skillopt-now`, `skillopt-eval`, `skillopt-new-skill`, `skillopt-train`
2. それらを Copilot CLI のスキル置き場 `~/.copilot/skills/` にコピーします。

PowerShell でまとめてコピーする例:

```powershell
# このガイドがあるフォルダで実行してください
$dest = "$env:USERPROFILE\.copilot\skills"
New-Item -ItemType Directory -Path $dest -Force | Out-Null

foreach ($s in @("skillopt-sleep","skillopt-now","skillopt-eval","skillopt-new-skill","skillopt-train")) {
    robocopy ".\$s" "$dest\$s" /E /XD __pycache__ | Out-Null
    Write-Host "コピー完了: $s"
}
```

3. **Copilot CLI を再起動**します（または `/clear` を実行）。
   - スキルは起動時に読み込まれるため、この操作で初めて認識されます。

> すでに `~/.copilot/skills/` に同名フォルダがある場合は上書きされます。心配な場合は事前にバックアップしてください。

## 3. 最初の動作確認

インストールが成功したかを、安全に確かめます（何も変更しません）。

```powershell
& "$env:USERPROFILE\.copilot\skills\skillopt-sleep\run.ps1" --proof
```

次のような行が最後に出れば成功です。

```text
baseline held-out : 0.3333
after  held-out   : 1.0   (lift +0.6667)
PASS: nightly consolidation improves held-out score AND gate blocks regressions.
```

これは「学習前の正答率 0.33 が、学習後に 1.0 へ上がり、悪い変更はきちんと却下された」という意味です。

## 4. スキルの使い方（共通の考え方）

スキルは 2 通りの方法で使えます。

### 方法 A: Copilot に話しかける（おすすめ）

再起動後、Copilot に自然な言葉で頼むと、対応するスキルが自動で起動します。

| 言い方の例 | 起動するスキル |
|---|---|
| 「sleep cycle を実行して」 | `skillopt-sleep` |
| 「今すぐスキルを最適化して」 | `skillopt-now` |
| 「この SKILL.md を評価して」 | `skillopt-eval` |
| 「コードレビュー用のスキルを作って」 | `skillopt-new-skill` |
| 「searchqa でベンチマーク学習して」 | `skillopt-train` |

### 方法 B: ランナーを直接実行する

スクリプトを自分で動かすこともできます（下の各スキル節を参照）。

### 大事な安全ルール

- **`adopt`（採用）するまで、あなたの SKILL.md は書き換わりません。**
- 採用時は必ずバックアップが作られます。
- `mock` モードは費用ゼロ。`claude` などの実モードは API 費用が発生します。

## 5. 各スキルの使い方

### 5-1. skillopt-sleep — 過去の作業から自動改善

過去のセッションを振り返り、繰り返し作業のコツをスキルに取り込みます。

```powershell
$SK = "$env:USERPROFILE\.copilot\skills\skillopt-sleep\run.ps1"

# 状態を見る（読み取りのみ）
& $SK status --project "$PWD"

# お試し実行（提案を作るが保存しない）
& $SK dry-run --project "$PWD" --backend mock --progress

# 本番実行（提案を保存。まだ SKILL.md は変えない）
& $SK run --project "$PWD" --backend mock --progress

# 提案を採用（バックアップ付きで反映）
& $SK adopt --project "$PWD"
```

毎晩の自動実行を設定/解除する場合:

```powershell
& $SK schedule --project "$PWD" --hour 3 --minute 17
& $SK unschedule --project "$PWD"
```

### 5-2. skillopt-now — 今すぐ 1 回だけ最適化

`sleep` の即時版。スケジュールを使わず、その場で 1 回実行します。

```powershell
$NOW = "$env:USERPROFILE\.copilot\skills\skillopt-now\run.ps1"
& $NOW run --project "$PWD" --backend mock --max-tasks 10 --progress
# 結果を確認して良ければ:
& $NOW adopt --project "$PWD"
```

### 5-3. skillopt-eval — スキルの品質をスコア化

スキルの「今の実力」を数値で測ります。学習はしません。

```powershell
$EV = "$env:USERPROFILE\.copilot\skills\skillopt-eval\run.ps1"

# Quick モード（自己完結・費用ゼロ）
& $EV -Mode quick -Project "$PWD" -Args @('--backend','mock','--max-tasks','20')

# Benchmark モード（公式スコア。リポジトリと依存が必要）
& $EV -Mode benchmark -Config "configs/searchqa/default.yaml" -Skill "C:\path\to\SKILL.md" -OutRoot "outputs/eval1"
```

### 5-4. skillopt-new-skill — スキルをゼロから作成

新しい用途のスキル（`SKILL.md`）を対話的に作ります。Copilot に話しかけて使うのが簡単です。

> 例: 「SQL クエリ生成用のスキルを作って。いつも LIMIT を付けるルールにしたい」

作成後、その場で `skillopt-now` による最適化を提案します（任意）。

### 5-5. skillopt-train — ベンチマークでフル学習（研究用）

公式ベンチマークで本格的に学習します。**このスキルだけは追加準備が必要**です。

```powershell
$TR = "$env:USERPROFILE\.copilot\skills\skillopt-train\run.ps1"

# 初回のみ: 依存パッケージをインストール
& $TR -Setup

# 使える設定ファイルの一覧
& $TR -ListConfigs

# 学習を実行（時間と API 費用がかかります）
& $TR -Config "configs/searchqa/default.yaml" -Args @('--num_epochs','2')
```

## 6. 困ったときは（トラブルシューティング）

| 症状 | 原因 | 対処 |
|---|---|---|
| Copilot がスキルを認識しない | 起動時に読み込まれる | Copilot CLI を再起動、または `/clear` |
| `need Python >= 3.10` と出る | Python が古い/未インストール | Python 3.10 以上を入れて `python --version` で確認 |
| `vendored engine not found` | 同梱エンジンが見つからない | `skillopt-sleep\sync-vendor.ps1` を実行して再同梱 |
| `0 sessions -> 0 tasks` になる | 取り込める履歴がない | 正常。動作確認は `--proof` を使う |
| train で `exit 2` / 依存不足 | パッケージ未インストール | `skillopt-train\run.ps1 -Setup` を実行 |
| train でリポジトリが見つからない | 本体リポジトリの場所が不明 | `-Repo "C:\path\to\SkillOpt"` を指定、または環境変数 `SKILLOPT_REPO` を設定 |

### 同梱エンジンを最新化したいとき

```powershell
& "$env:USERPROFILE\.copilot\skills\skillopt-sleep\sync-vendor.ps1" -Repo "C:\path\to\SkillOpt"
```

`vendor\VERSION.txt` に取り込み元のコミット情報が記録されます。

## 7. 用語集

| 用語 | 意味 |
|---|---|
| スキル / SKILL.md | Copilot CLI に機能を足す仕組みと、その本体テキスト。 |
| backend（バックエンド） | 実際に処理する担当。`mock`=費用ゼロのお試し、`claude`/`codex`/`copilot`=実モデル。 |
| held-out スコア | 学習に使わず取り置いた問題での正答率。改善の良し悪しを公平に測る指標。 |
| gate（ゲート） | held-out スコアが上がった変更だけを通す検問。悪化を防ぐ安全装置。 |
| stage（ステージ） | 提案をいったん保存すること。まだ本番ファイルは変えない。 |
| adopt（採用） | 提案を本番の SKILL.md に反映すること。バックアップを取ってから行う。 |
| vendor（同梱） | 外部に頼らず、必要なコードをスキル内に持つこと。 |
| PYTHONPATH | Python がコードを探す場所の設定。ランナーが自動で設定する。 |
